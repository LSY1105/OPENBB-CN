技术背景：Transformer 架构

Foundations · Attention Mechanism
🔬 核心创新：多头自注意力
Attention(Q,K,V) = softmax(QK^T / √d_k) × V MultiHead = Concat(head_1,...,head_h) × W^O 其中 head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
  • 多头并行：h=8 个注意力头，d_k=64
  • 缩放因子 √d_k 防止梯度饱和
  • 路径长度 O(1) vs RNN 的 O(n)
📊 位置编码与架构配置
PE(pos,2i) = sin(pos / 10000^(2i/d_model)) PE(pos,2i+1) = cos(pos / 10000^(2i/d_model)) BERT_BASE: L=12, H=768, A=12, d_ff=2048, 110M参数 BERT_LARGE: L=24, H=1024, A=16, d_ff=4096, 340M参数
  • 正弦编码可泛化到更长序列
  • 残差连接 + LayerNorm 稳定训练
  • 位置编码使 Transformer 超越 RNN 的局限
🧠 BERT（Encoder）
双向理解 · MLM预训练 · GLUE刷榜 +7.7%
⚡ GPT（Decoder）
单向生成 · In-Context Learning · 千亿参数
🎯 MoE（稀疏激活）
Mixtral: 46.7B总参/13B活跃 · Top-K=2路由
BERT催生预训练-微调范式，MoE打破参数规模与计算成本线性关系——OpenClaw正站在这两大突破之上。

安全加固（v4.2）

Enterprise Security · Supply Chain Protection
🚨 插件危险代码扫描
背景：Cisco AI 安全研究团队发现第三方 OpenClaw 技能存在数据外泄和提示注入风险。
  • 内置静态分析，检测到严重问题时安装直接失败
  • 绕过需显式 --dangerously-force-unsafe-install
  • 发布到 ClawHub 的技能必须通过危险代码扫描
🔐 节点配对权限分离
  • 旧逻辑：配对成功 → 自动获得节点命令执行权限
  • 新逻辑：配对成功 + 明确批准 → 才能执行节点命令
  • 防止配对流程被滥用来提权攻击
🛡️ Shell 环境变量过滤
# 以下环境变量禁止从请求作用域传入: PYPI_INDEX / pip_proxy # 包拉取重定向 DOCKER_HOST / CONTAINERD_HOST # 容器劫持 TLS_CERT_PATH / SSL_CERT_DIR # 证书伪造 GCC_PATH / CLANG_PATH # 编译器注入 MAVEN_OPTS / GRADLE_OPTS # JVM依赖劫持
封堵：通过注入环境变量将包拉取重定向到恶意源的供应链攻击
🔒 其他加固项
  • 传输层 TLS 集中化 — HTTP/WebSocket/流式传输统一认证
  • SSRF 防护 — 停止从图像 base URL 推断私网访问
  • Dotenv 覆盖保护 — workspace .env 不能覆盖 OPENCLAW_PINNED_*
  • Webhook 安全比较 — Feishu/Telegram 等统一用 safeEqualSecret
