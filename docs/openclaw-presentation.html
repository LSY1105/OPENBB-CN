什么是 OpenClaw？

What is OpenClaw
OpenClaw 是一个生产级的多 Agent AI 协作平台。一个实例支持多个 AI Agent 协同工作，每个 Agent 拥有独立的人设、技能与工作区。通过灵活的工具链集成和持久化知识库，实现跨渠道、跨任务、跨会话的智能协作。
🤖 多 Agent 系统
多个 AI Agent 各司其职、协同作战，通过 binding 配置区分不同 Agent 的能力边界与专业领域。
⚡ 智能任务流
SQLite 持久化的任务账本，48 小时超长待机，统一调度 ACP 任务、Cron 定时任务与子 Agent。
🧠 持久知识库
Karpathy 模式的四阶段循环：Ingest → Compile → Query → Lint，知识编译一次，持续复利。
🔒 企业级安全
危险代码静态扫描、环境变量严格过滤、节点权限分离，保障系统安全可靠。

Task Flow 任务流

SQLite · /tasks · ACP · Sticky Cancel
⚡ 核心特性
  • SQLite 持久化 — 任务状态持久存储，断电不丢
  • 修订版追踪 — Managed vs Mirrored 双模式
  • /tasks 命令 — 聊天窗口实时查看所有后台任务
  • Sticky Cancel Intent — 取消时等子任务完成再停止
  • 阻塞可见 — 任务被阻塞时父级会话直接显示原因
  • api.runtime.taskFlow — 插件可直接创建和管理任务流
🧠 与 Karpathy 知识库的关系
Task Flow 管理主动任务的执行，
Karpathy 知识库管理被动知识的积累。

两者共同构成 OpenClaw 的时间维度与空间维度——任务解决当下问题，知识编译持续复利。
🔄 三类任务统一调度
类型说明超时
ACP主动调用 /task 触发默认48h
Cron定时任务 · 信号采集 · 每日复盘独立配置
Subagent子智能体 · spawn 隔离会话继承+扩展

安全加固（v4.2）

Enterprise Security
🚨 插件危险代码扫描
背景：Cisco AI 安全研究团队发现第三方 OpenClaw 技能存在数据外泄和提示注入风险。
  • 内置静态分析，检测到严重问题时安装直接失败
  • 绕过需显式 --dangerously-force-unsafe-install
  • 发布到 ClawHub 的技能必须通过危险代码扫描
🔐 节点配对权限分离
  • 旧逻辑：配对成功 → 自动获得节点命令权限
  • 新逻辑：配对成功 + 明确批准 → 才能执行节点命令
  • 防止配对流程被滥用来提权攻击
🛡️ Shell 环境变量严格过滤
# 以下环境变量禁止从请求作用域传入: PYPI_INDEX / pip_proxy # 包拉取重定向 DOCKER_HOST / CONTAINERD_HOST # 容器劫持 TLS_CERT_PATH / SSL_CERT_DIR # 证书伪造 GCC_PATH / CLANG_PATH # 编译器注入 MAVEN_OPTS / GRADLE_OPTS # JVM依赖劫持
🔒 其他加固项
  • 传输层 TLS 集中化 — HTTP/WebSocket/流式统一认证
  • SSRF 防护 — 停止从图像 URL 推断私网访问
  • Dotenv 覆盖保护 — workspace .env 不能覆盖 OPENCLAW_PINNED_*
  • Webhook 安全比较 — Feishu/Telegram 等统一用 safeEqualSecret
Cisco AI 安全研究 · SSRF 防护 · 传输层 TLS 集中化
